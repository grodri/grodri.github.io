Bundling Image Files in a Web Page using base64

Many Stata tools, including my own markstat command, can generate web pages that include images. Usually, however, the resulting document is not self-contained because all it has are links to the images.

The bundle command solves this problem by going through the document, grabbing all images in PNG format, and encoding each of them as text using base64, the same format as email attachments.

The syntax of the command is

bundle input_filename [, output_filename]

The input_filename argument is required and must conform to Stata conventions for filenames. The extension may be omitted if it is .html

The output_filename argument is optional. If omitted the command will create a file with the same name as the input, plus the postfix -b for bundle, and the same extension. If the file name is specified the extension may be omitted and will default to .html. If the file exists it will be overwritten.

The command lists each image file as it is processed, and writes the output file name when it is saved. If no image files are found no output is generated.

Installation

The command is available from the Statistical Software Components (SSC) archive and can be installed by typing in Stata 

. ssc install bundle

You may also try search bundle and follow the links.
